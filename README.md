# Сайт для консалтинговой компании

Этот проект представляет собой корпоративный сайт, разработанный для консалтинговой компании. Он включает в себя информацию о компании, ее услугах, команде, а также блог со статьями.

## Технологический стек

*   **Фреймворк:** [Next.js](https://nextjs.org/) 15
*   **Язык:** [TypeScript](https://www.typescriptlang.org/)
*   **UI библиотека:** [React](https://reactjs.org/) 18
*   **Стилизация:** [Tailwind CSS](https://tailwindcss.com/)
*   **3D графика:** [Three.js](https://threejs.org/) / [React Three Fiber](https://docs.pmnd.rs/react-three-fiber/getting-started/introduction)
*   **Анимации:** [Framer Motion](https://www.framer.com/motion/)
*   **Менеджер пакетов:** [pnpm](https://pnpm.io/)

## Требования к окружению

*   **Node.js:** версия 20.x
*   **pnpm:** версия 10.x или выше

## Установка и запуск

1.  **Клонируйте репозиторий:**
    ```bash
    git clone <URL репозитория>
    cd pc-site
    ```

2.  **Установите зависимости:**
    ```bash
    pnpm install
    ```

3.  **Настройте переменные окружения:**
    Создайте файл `.env.local` в корне проекта и добавьте в него переменные, описанные в секции «Переменные окружения».

4.  **Запустите проект в режиме разработки:**
    ```bash
    pnpm dev
    ```
    После этого сайт будет доступен по адресу [http://localhost:3000](http://localhost:3000).

## Переменные окружения

Для корректной работы формы обратной связи необходимо настроить интеграцию с Google Sheets.

Создайте файл `.env.local` в корневой директории проекта и добавьте в него следующие переменные:

```
# ID вашей Google Таблицы. Его можно найти в URL: 
# https://docs.google.com/spreadsheets/d/THIS_IS_THE_ID/edit
GOOGLE_SHEET_ID="ВАШ_ID_ТАБЛИЦЫ"

# Учетные данные сервисного аккаунта Google в формате Base64.
GOOGLE_CREDENTIALS_BASE64="ВАШИ_УЧЕТНЫЕ_ДАННЫЕ_В_BASE64"
```

### Как получить `GOOGLE_CREDENTIALS_BASE64`:

1.  Создайте сервисный аккаунт в [Google Cloud Console](https://console.cloud.google.com/).
2.  Скачайте JSON-файл с ключами для этого аккаунта.
3.  **Не добавляйте этот JSON-файл в репозиторий!**
4.  Сконвертируйте содержимое этого JSON-файла в строку Base64. Это можно сделать с помощью онлайн-конвертеров или командой в терминале:
    ```bash
    # Для macOS
    cat /путь/к/вашему/файлу.json | base64

    # Для Linux
    cat /путь/к/вашему/файлу.json | base64 -w 0
    ```
5.  Вставьте полученную строку в переменную `GOOGLE_CREDENTIALS_BASE64`.
6.  Не забудьте дать вашему сервисному аккаунту (по его email) права на редактирование вашей Google Таблицы.

## Основные команды

*   `pnpm dev`: запуск сервера для разработки.
*   `pnpm build`: сборка production-версии проекта.
*   `pnpm start`: запуск production-сборки.
*   `pnpm lint`: проверка кода на соответствие стандартам.
*   `pnpm test`: запуск тестов (если они настроены).

## Структура проекта

*   `src/app/`: Основная директория с страницами и роутингом Next.js.
    *   `src/app/api/`: API-роуты.
    *   `src/app/(pages)/`: Группы роутов для основных страниц (о нас, услуги, проекты и т.д.).
*   `src/components/`: Переиспользуемые React-компоненты.
    *   `src/components/layout/`: Компоненты макета (Header, Footer).
    *   `src/components/cards/`: Карточки для статей, проектов и т.д.
*   `src/lib/`: Вспомогательные функции и утилиты.
*   `public/`: Статические файлы (изображения, шрифты, документы).
*   `next.config.ts`: Конфигурационный файл Next.js.
*   `tailwind.config.ts`: Конфигурационный файл Tailwind CSS.
